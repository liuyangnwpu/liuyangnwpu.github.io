Equation

Normally, for inlineMath, $,$ or \\(,\\) is used, while for displayMath, $$,$$ or \[,\] is used. However, $ is not recommended for its common use in plain text, so \\(,\\) and \[,\] are better choice.

The different interpretation of the underscore character _ by Markdown and Mathjax introduces an issue with equations. After some tests, I think the best way for writing equations are:

(1) Display equation

Use div block so I can directly copy latex form equation from Mathtype, otherwise, I have to change _ to \_ to make the subscript right.

<div> \[{{\bf{K}}_k} = {{\bf{P}}_{k/k - 1}}{{\bf{H}}_k}^T{\left( {{{\bf{H}}_k}{{\bf{P}}_{k/k - 1}}{{\bf{H}}_k}^T + {{\bf{R}}_k}} \right)^{ - 1}}\]</div>
\[{{\bf{K}}_k} = {{\bf{P}}_{k/k - 1}}{{\bf{H}}_k}^T{\left( {{{\bf{H}}_k}{{\bf{P}}_{k/k - 1}}{{\bf{H}}_k}^T + {{\bf{R}}_k}} \right)^{ - 1}}\]

(2) Inline equation

Inline equation \\({t\_{k - 1}} + i\Delta t\\) is OK.

Inline equation \({t_{k - 1}} + i\Delta t\) is OK. For inline equations, as it is often short, change _ to \_ is acceptable.

Sometime I find that directly use _ is fine in inline equations, but the use of \_ is more reliable. Ref

Blackburn Figures

Short Code:

{{< fluid_imgs

“class|src|alt”
“class|src|alt”
“… and so on”
>}}

where each positional parameter is separated by the vertical bar (i.e., |).

  • class: specifies a Pure CSS unit class name (required)
  • src: specifies the URL of an image (required)
  • alt: specifies an alternate text for an image (optional)

The shortcode uses positional parameters so the order is important. The first two parameters must be specified in that order. Also see the official Pure CSS site for valid unit class names.

Example:

{{< fluid_imgs
“pure-u-1-3|/img/xian.jpg”
“pure-u-1-3|/img/xian.jpg”
“pure-u-1-3|/img/xian.jpg”
>}}
